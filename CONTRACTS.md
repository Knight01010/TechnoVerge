## Rules (all workers)
- Every component is a client component: first line `'use client';`.
- One folder: components/`<Name>.tsx` + components/`<Name>.module.css`. Touch ONLY your assigned files. Do not edit layout, globals.css, page.tsx, providers, lib/*.
- All copy/data comes from `@/lib/content` (already written — read it). Never hardcode copy that exists there.
- Global CSS tokens available: `--bg --panel --panel-alt --ink --ink-dim --ink-faint --line --red --red-dim --font-display --font-mono --ease-out --ease-inout --pad-x`. Global keyframes: `blink pulseDot marquee spinSlow dashMove menuScan`. Use them; don't redeclare.
- Shared context: `useApp()` from `@/components/providers/AppProviders`:
  `{ lenisRef, menuOpen, toggleMenu, closeMenu, loaderDone, setLoaderDone, navigateTo(hash), reducedMotion }`.
  - `lenisRef.current` may be **null** (before mount / reduced motion) — guard.
  - Anchor `<a href="#...">` clicks are globally intercepted by the provider and run the red curtain transition — plain anchors just work; don't add your own smooth-scroll.
- Scramble text util: `scramble(el, finalText, durationMs, onDone?)` from `@/lib/scramble` — returns cancel fn; cancel in cleanup.
- GSAP: `import { gsap } from 'gsap'; import { ScrollTrigger } from 'gsap/ScrollTrigger';` and call `gsap.registerPlugin(ScrollTrigger)` inside your effect (safe, idempotent). Create animations inside `gsap.context(() => {...}, rootRef)` (or `useGSAP` pattern manually) and `ctx.revert()` in cleanup. Never leak ScrollTriggers.
- Respect `reducedMotion` from `useApp()`: skip scrambles/parallax/scrub; content must be fully visible and readable without JS-driven animation.
- Hooks rules strictly: no conditional hooks, cleanup every effect (intervals, rafs, listeners, observers), no setState after unmount.
- Custom-cursor labels: put `data-cursor="LABEL"` on interactive elements that should enlarge the cursor ring with a label (e.g. `data-cursor="TALK"`, `data-cursor="GO"`, `data-cursor="+"`, empty string for plain grow).
- Images: `next/image` with explicit width/height; decorative images `alt=""` + `aria-hidden` where wrapped. Logos: `/assets/logo-mark.png` (red mark, 95×119) and `/assets/logo-mark-black.png` (93×118).
- Section shells must keep ids: `hero, manifesto, practices, services, process, contact` (used by menu + footer links).
- A11y: semantic headings (single h1 in Hero), buttons with aria labels, decorative HUD text `aria-hidden` where it's noise for screen readers.
- TypeScript strict — no `any` unless unavoidable; must pass `npm run build` (typecheck + ESLint).
